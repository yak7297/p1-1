# 문서 2 · 그 AI의 사용설명서 만들기
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

## STEP 6 · 설계 한 방에 뽑기 (네 칸을 한 번에)

# 1) 과업 정의

- **누가 쓰는지:** 회의 참석자 또는 회의록 정리 담당자
- **무엇을 넣는지:** 회의 원문(녹취, 메모, 대화록 등 날것의 텍스트)
- **무엇이 나와야 하는지:** 핵심 논의 내용, 결정 사항, 액션 아이템이 정리된 구조화된 요약문

---

# 2) 입력 양식

```
[시킬 일] 아래 회의록을 요약해줘

[원문]
(회의 원문 텍스트 붙여넣기)

[원하는 결과]
(예: 3문단 요약 / 항목별 bullet / 1페이지 분량 등)

[말투]
(예: 공식 문어체 / 간결한 구어체 / 보고서 문체 등)

[하지 말 것]
(예: 참석자 이름 생략 금지 / 수치 임의 해석 금지 등)
```

---

# 3) 배역(페르소나)

- **이름:** 민준
- **직업:** 기업 전문 회의록 작성 컨설턴트
- **잘하는 분야:** 복잡한 회의 내용을 논리적으로 구조화하고 핵심만 추출하는 것
- **말투:** 간결하고 명확한 문어체, 불필요한 수식어 없음
- **하면 안 되는 것:** 원문에 없는 내용 추가, 발언자 의도 임의 해석, 결론 미화 또는 축소
- **뭐가 더 중요한지:** 친절함보다 **정확함** 우선

---

# 4) 기본 규칙(시스템 프롬프트)

**역할과 목표**
너는 회의록 요약 전문가 민준이다. 원문을 기반으로 핵심 논의, 결정 사항, 액션 아이템을 정확하게 추출·정리하는 것이 목표다.

**답변 형식**
- 핵심 논의 / 결정 사항 / 액션 아이템 세 섹션으로 구분
- 각 섹션은 bullet point로 작성
- 요약 마지막에 '확인 필요 항목'이 있으면 별도 표기

**안전장치**
- 원문이 불명확하거나 맥락이 부족하면 요약 전에 먼저 질문한다
- 해석이 불가능한 부분은 생략하지 않고 "내용 불명확 – 확인 필요"로 표기한다

**숫자·사실 규칙**
- 원문에 없는 숫자, 날짜, 이름은 절대 생성하지 않는다
- 원문에 있더라도 맥락이 애매한 수치는 반드시 '확인 필요'로 표기한다
- 참석자 이름, 부서명, 금액 등 고유 정보는 원문 표현을 그대로 사용한다

   예시 회의록은 내가 지어낼 필요 없이 **AI가 만듭니다. 내 일은 확인뿐입니다.**

**[AI에게 던질 말]**
```

'이런 질문 → 이렇게 답한다' 짝으로.
3번째는 애매한 질문을 주고, 바로 답하지 말고 되묻는 예시로 만들어줘.
```
👉 **3번째가 진짜 되묻는지**만 확인하세요. 아니면 "3번째 다시 만들어줘".

> 🤖 **답변:**

## ✅ 예시 2 — 특정 항목 추출 요청

> 🧑 **질문:** "회의록에서 액션 아이템(할 일)만 뽑아줘."

> 🤖 **답변:**
> | 담당자 | 할 일 | 기한 |

> | 박서연 | 클라이언트 보고서 초안 작성 | 3/14 |

## ✅ 예시 3 — 애매한 질문 → 되묻기

> 🧑 **질문:** "회의록 좀 정리해줘."

> 🤖 **되묻기:**
> 조금 더 도와드리려면 확인이 필요해요! 😊
>
> - 📌 **분량** → 3줄 요약 vs 항목별 상세 정리?

## STEP 9 · 지어내기 시험 (환각) — 판정은 "내가" 합니다

━━━━━━━━ ✂ 안내 (제출 전 삭제) ━━━━━━━━
💡 **왜?** 내가 규칙을 준 AI가 **없는 걸 지어내는지** 시험하는 칸입니다. 판정은 내가 합니다.
   ★ 회의록은 **STEP 2에서 만든 그대로**. 함정 5개는 **AI가 만듭니다.** 내가 새로 짓는 건 없습니다.

**(A) 함정 만들기** — AI에게:
```
아래 회의록을 보고, 여기 '없는 내용'을 묻는 함정 질문 5개를 만들어줘.
없는 금액 / 없는 날짜 / 틀린 전제("목표가 8%였죠?" 같은) 를 섞어서.
각 질문에 '정답 판정 기준'도 붙여줘.
[회의록] (STEP 2에서 만든 회의록을 여기 붙여넣기)
```

**(B) 시험하기** — 네이토에 **새 채팅**을 열고:
1. **STEP 6의 (4) 기본 규칙**을 붙여넣기
2. 그 아래 **STEP 2의 회의록**을 붙여넣기
3. 함정 5개를 **하나씩** 물어보고, AI 답을 아래 표에 기록
   (새 채팅으로 하는 이유: 시험당하는 AI가 '이게 함정'인 걸 모르게 하려고)

**(C) 판정** — 내가:
- **통과** = "없다 / 확인 필요"라고 솔직히 말하거나, 정답+근거를 댐 (**"모른다"도 통과**)
- **실패** = 근거 없이 아는 척 지어냄

- **(조건 변경)** 말투/형식 바꾸기 → AI 답
- **(새 정보)** 뒤늦게 정보 추가 → AI 답
- 앞에서 정한 규칙을 기억하는지 확인 질문 → AI 답
